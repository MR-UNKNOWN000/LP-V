# 2. What is DFS?

DFS (Depth First Search) visits nodes by going **as deep as possible first**, then backtracks.

### Example Graph

```text
      0
     / \
    1   2
    |   |
    3   4
```

Starting from `0`, one possible DFS traversal is:

```text
0 1 3 2 4
```

---

# 3. Real-Life Example

Imagine exploring rooms in a building:

* Enter one room.
* Keep moving to connected rooms until no further room is available.
* Then come back and try another path.

This is DFS.

# 8. Sequential DFS

```cpp
void dfsSequential(int start)
```

Traverses the graph one node at a time using a stack.

---

## Step 1: Create visited array and stack

```cpp
vector<bool> visited(n, false);
stack<int> s;
```

* `visited` keeps track of visited nodes.
* `s` stores nodes to be processed.

---

## Step 2: Push Start Node

```cpp
s.push(start);
```

Adds the starting node to the stack.

---

## Step 3: Process Until Stack Is Empty

```cpp
while (!s.empty())
```

Continue until all reachable nodes are visited.

---

## Step 4: Remove Top Node

```cpp
int node = s.top();
s.pop();
```

Take the top node from the stack.

---

## Step 5: Check if Already Visited

```cpp
if (!visited[node])
```

Process the node only once.

---

## Step 6: Mark and Print

```cpp
visited[node] = true;
cout << node << " ";
```

---

## Step 7: Push Unvisited Neighbors

```cpp
for (int i = adj[node].size() - 1; i >= 0; i--)
```

Reverse order is used so traversal appears natural.

```cpp
if (!visited[next]) {
    s.push(next);
}
```

---

# 9. Why Reverse Traversal?

Stack is LIFO (Last In First Out).

If neighbors are `1` and `2`:

* Push `2` first
* Push `1` next

Then `1` is processed before `2`.

# 18. Time Complexity

DFS visits every vertex and edge once.

O(V + E)

Where:

* `V` = number of vertices
* `E` = number of edges

---

# 19. Space Complexity

* Adjacency list
* Visited array
* Stack

O(V + E)

# 21. Why Stack Is Used

DFS follows LIFO:

* Last inserted node is processed first.

---

# 22. BFS vs DFS

| BFS                                      | DFS                       |
| ---------------------------------------- | ------------------------- |
| Uses queue                               | Uses stack                |
| Level by level                           | Depth first               |
| Finds shortest path in unweighted graphs | Good for path exploration |

---

# 23. Sequential vs Parallel DFS

| Sequential         | Parallel                          |
| ------------------ | --------------------------------- |
| One node at a time | Multiple branches can be explored |
| Single thread      | Multiple threads possible         |
| Simpler            | Faster for large graphs           |

# 24. Viva Questions and Answers

### Q1. What is DFS?

DFS is a graph traversal method that explores one path completely before backtracking.

### Q2. Which data structure is used?

Stack.

### Q3. Why is visited array used?

To avoid revisiting nodes.

### Q4. What is the time complexity?

O(V + E).

### Q5. What is the space complexity?

O(V + E).

### Q6. What does `top()` do?

Returns the top element of the stack.

### Q7. What does `pop()` do?

Removes the top element.

### Q8. What does `push()` do?

Adds an element to the stack.

### Q9. What is OpenMP?

A library for parallel programming.

### Q10. What does `omp_get_wtime()` do?

Returns the current wall-clock time.

### Q11. Why are neighbors pushed in reverse order?

To maintain the expected DFS traversal order.

### Q12. What happens if visited array is removed?

The program may revisit nodes and loop indefinitely.

### Q13. Can DFS detect cycles?

Yes.

### Q14. What is an undirected graph?

A graph where edges work in both directions.

### Q15. What is backtracking?

Returning to a previous node when no unvisited neighbors remain.

# 26. Short Summary to Remember

* Graph stored as adjacency list.
* Start node is pushed onto the stack.
* Pop top node.
* If unvisited, mark and print it.
* Push all unvisited neighbors.
* Repeat until stack is empty.
