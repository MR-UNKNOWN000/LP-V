# 1. Aim of the Program

To find:

1. Minimum value
2. Maximum value
3. Sum of all elements
4. Average of all elements

using:

* Sequential approach
* Parallel reduction concept using OpenMP

---

# 2. What is Reduction?

Reduction means combining many values into one result.

Examples:

* Many numbers → one minimum
* Many numbers → one maximum
* Many numbers → one sum
* Many numbers → one average

# 7. Parallel Functions

These functions are conceptually the same as sequential ones.

In a full OpenMP implementation, the loops would use reduction clauses such as:

```cpp id="vbg6gd"
#pragma omp parallel for reduction(min:mn)
#pragma omp parallel for reduction(max:mx)
#pragma omp parallel for reduction(+:sum)
```

Your simplified code keeps the logic straightforward for exam use.

---

Finds the minimum.

## findMaxParallel()

## findSumParallel()

## findAverageParallel()

Finds the average using the parallel sum.

# 15. Time Complexity

Each operation scans the entire array once.

O(n)

Where `n` is the number of elements.

---

# 16. Space Complexity

O(1)

Only a few variables are used.

(The vector storing input uses O(n).)

---

# 17. Why `long long` Is Used for Sum

The sum may exceed the range of `int`.

Example:

* 100000 numbers × large values

`long long` prevents overflow.

---

# 18. Why Average Returns `double`

Average may contain decimal values.

Example:
7 / 2 = 3.5

---

# 19. Sequential vs Parallel Reduction

| Sequential                   | Parallel                                          |
| ---------------------------- | ------------------------------------------------- |
| One element at a time        | Multiple elements can be processed simultaneously |
| Single thread                | Multiple threads possible                         |
| Slower for very large arrays | Faster for large arrays                           |
