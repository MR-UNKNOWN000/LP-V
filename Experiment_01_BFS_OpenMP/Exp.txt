# Parallel BFS using OpenMP – Complete Practical Explanation

---

# 1. Aim of the Program

To perform **Breadth First Search (BFS)** on a graph using:

1. Sequential approach
2. Parallel approach using OpenMP

---

# 2. What is BFS?

BFS (Breadth First Search) visits nodes **level by level**.

### Example Graph

```text
0
| \
1  2
|   \
3    4
```

Starting from `0`, BFS traversal is:

```text
0 1 2 3 4
```

---

# 3. Real-Life Example

Imagine exploring friends on social media:

* First, you see your direct friends.
* Then, friends of your friends.
* Then, next level.

This is exactly BFS.

# 7. addEdge()

```cpp
void addEdge(int u, int v) {
    adj[u].push_back(v);
    adj[v].push_back(u);
}
```

### Meaning

Adds an undirected edge between `u` and `v`.

If input is:

```text
0 1
```

Then:

* 1 is added to adjacency list of 0
* 0 is added to adjacency list of 1

# 8. Sequential BFS

```cpp
void bfsSequential(int start)
```

Traverses the graph one node at a time.

---

## Step 1: Create visited array and queue

```cpp
vector<bool> visited(n, false);
queue<int> q;
```

* `visited` keeps track of visited nodes.
* `q` stores nodes to be processed.

---

## Step 2: Start Node

```cpp
visited[start] = true;
q.push(start);
```

Marks the starting node as visited and adds it to the queue.

---

## Step 3: Process Until Queue Is Empty

```cpp
while (!q.empty())
```

Continue until all reachable nodes are visited.

---

## Step 4: Remove Front Node

```cpp
int node = q.front();
q.pop();
```

Take the first node from the queue.

---

## Step 5: Print Node

```cpp
cout << node << " ";
```

Display the visited node.

---

## Step 6: Visit Neighbors

```cpp
for (int i = 0; i < adj[node].size(); i++)
```

Loop through all neighbors.

---

## Step 7: Add Unvisited Neighbor

```cpp
if (!visited[next]) {
    visited[next] = true;
    q.push(next);
}
```

Mark and enqueue each unvisited neighbor.

---

# 9. Parallel BFS

```cpp
void bfsParallel(int start)
```

Works level by level and is intended to be parallelized.

---

## Step 1: Remove All Nodes of Current Level

```cpp
int size = q.size();
vector<int> level;
```

Stores all nodes currently in the queue.

---

## Step 2: Move Current Level into `level`

```cpp
for (int i = 0; i < size; i++) {
    int node = q.front();
    q.pop();
    level.push_back(node);
}
```

---

## Step 3: Print Current Level

```cpp
for (int i = 0; i < level.size(); i++) {
    cout << level[i] << " ";
}
```

---

## Step 4: Explore Neighbors

```cpp
for (int i = 0; i < level.size(); i++)
```

Each node in the current level is processed.

> Note: In your simplified exam code, this loop is sequential. In a true OpenMP version, you would add:

```cpp
#pragma omp parallel for
```

# 16. Dry Run

Queue starts with:

```text
[0]
```

Visit `0`, add `1` and `2`.

Queue:

```text
[1, 2]
```

Visit `1`, add `3`.

Queue:

```text
[2, 3]
```

Visit `2`, add `4`.

Queue:

```text
[3, 4]
```

Visit `3`, then `4`.

Traversal:

```text
0 1 2 3 4
```

# 17. Time Complexity

BFS visits every vertex and edge once.

O(V + E)

Where:

* `V` = number of vertices
* `E` = number of edges

---

# 18. Space Complexity

* Adjacency list
* Visited array
* Queue

O(V + E)

---

# 19. Important Data Structures

| Data Structure        | Purpose              |
| --------------------- | -------------------- |
| `vector<vector<int>>` | Stores graph         |
| `queue<int>`          | Maintains BFS order  |
| `vector<bool>`        | Tracks visited nodes |

---

# 20. Why Queue Is Used

BFS follows FIFO:

* First inserted node is processed first.

---

# 21. Sequential vs Parallel BFS

| Sequential                   | Parallel                       |
| ---------------------------- | ------------------------------ |
| Processes one node at a time | Processes nodes level by level |
| Single thread                | Multiple threads possible      |
| Simpler                      | Faster for large graphs        |

# 22. Viva Questions and Answers

### Q1. What is BFS?

BFS is a graph traversal method that visits nodes level by level.

### Q2. Which data structure is used?

Queue.

### Q3. Why is visited array used?

To avoid visiting the same node multiple times.

### Q4. What is the time complexity?

O(V + E).

### Q5. What is the space complexity?

O(V + E).

### Q6. What does `adj` store?

Adjacency list.

### Q7. What does `push()` do?

Adds an element to the queue.

### Q8. What does `pop()` do?

Removes the front element.

### Q9. What does `front()` do?

Returns the first element of the queue.

### Q10. What is OpenMP?

A library used for parallel programming in C/C++.

### Q11. What does `omp_get_wtime()` do?

Returns the current wall-clock time.

### Q12. What is an undirected graph?

A graph where edges work in both directions.

### Q13. Why use queue instead of stack?

Queue is needed for level-order traversal.

### Q14. Can BFS find the shortest path?

Yes, in an unweighted graph.

### Q15. What happens if visited array is removed?

Nodes may be processed repeatedly, causing incorrect behavior.
